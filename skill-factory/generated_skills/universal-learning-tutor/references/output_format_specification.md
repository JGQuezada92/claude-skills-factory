## Format Requirements

### Markdown Standards

1. **Headers**: Use proper hierarchy (# for title, ## for major sections, ### for subtopics)
2. **Lists**: Use `-` for unordered, numbers for ordered
3. **Emphasis**: Use `**bold**` for key terms, `*italic*` for reasoning/notes
4. **Code blocks**: Use ``` for any code examples
5. **Horizontal rules**: Use `---` to separate major sections

### Content Standards

1. **Length**: 8,000-15,000 words total
2. **Subtopics**: 5-8 major subtopics
3. **Resources**: 10-15 ranked resources
4. **Quiz**: Exactly 10 questions
5. **Projects**: 3-5 hands-on prompts
6. **Examples**: 2-3 worked examples per subtopic
7. **Pitfalls**: 2-4 per subtopic
8. **Spaced repetition**: 4 prompts per subtopic

### Audio Optimization

1. **Conversational tone**: Write as if speaking to listener
2. **Clear transitions**: Use "First...", "Next...", "Now...", "Finally..."
3. **Sentence length**: Average 15-20 words
4. **Paragraph length**: 3-5 sentences maximum
5. **Avoid**: Tables, complex formatting, excessive bullets

### Visual Optimization

1. **White space**: Adequate spacing between sections
2. **Scannable**: Clear headers make it easy to find content
3. **Consistent formatting**: Maintain exact structure across all guides
4. **Emphasis**: Bold for key terms, italic for notes

## Quality Checklist

Before finalizing any learning guide, verify:

- [ ] Title and executive summary present
- [ ] 5-7 learning objectives listed
- [ ] 10-15 resources ranked with all required fields
- [ ] Resources categorized into progression path
- [ ] Knowledge architecture covers all subtopics
- [ ] Pre-assessment quiz has 10 questions
- [ ] 3-5 project prompts included
- [ ] Each subtopic has complete structure
- [ ] Core definitions are clear and quotable
- [ ] 3-5 key concepts per subtopic
- [ ] Worked examples show complete reasoning
- [ ] Common pitfalls identified
- [ ] Expert insights provided
- [ ] 4 spaced repetition prompts per subtopic
- [ ] Next steps are specific and actionable
- [ ] Glossary has 15-25 terms
- [ ] Format is audio-friendly
- [ ] Format is visually scannable
- [ ] All URLs are valid
- [ ] Content is factually accurate

## Adaptation by Domain

While structure remains constant, content adapts:

**Technical subjects**: More worked examples, precise definitions, mathematical rigor
**Business subjects**: More frameworks, case studies, decision criteria
**Creative subjects**: More examples from masters, process-focused, experimentation encouraged
**Sciences**: Multi-scale explanations, mechanisms, experimental evidence
**Humanities**: Multiple perspectives, context-heavy, critical analysis

The template accommodates all domains while maintaining standardization.
